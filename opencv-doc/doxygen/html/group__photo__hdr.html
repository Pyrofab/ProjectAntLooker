class  org.bytedeco.javacpp.opencv_photo.Tonemap
 Base class for tonemapping algorithms - tools that are used to map HDR image to 8-bit range.
 
class  org.bytedeco.javacpp.opencv_photo.TonemapDrago
 Adaptive logarithmic mapping is a fast global tonemapping algorithm that scales the image in logarithmic domain.
 
class  org.bytedeco.javacpp.opencv_photo.TonemapDurand
 This algorithm decomposes image into two layers: base layer and detail layer using bilateral filter and compresses contrast of the base layer thus preserving all the details.
 
class  org.bytedeco.javacpp.opencv_photo.TonemapReinhard
 This is a global tonemapping operator that models human visual system.
 
class  org.bytedeco.javacpp.opencv_photo.TonemapMantiuk
 This algorithm transforms image to contrast using gradients on all levels of gaussian pyramid, transforms contrast values to HVS response and scales the response. After this the image is reconstructed from new contrast values.
 
class  org.bytedeco.javacpp.opencv_photo.AlignExposures
 The base class for algorithms that align images of the same scene with different exposures.
 
class  org.bytedeco.javacpp.opencv_photo.AlignMTB
 This algorithm converts images to median fr.antproject.utils.threshold bitmaps (1 for pixels brighter than median luminance and 0 otherwise) and than aligns the resulting bitmaps using bit operations.
 
class  org.bytedeco.javacpp.opencv_photo.CalibrateCRF
 The base class for camera response calibration algorithms.
 
class  org.bytedeco.javacpp.opencv_photo.CalibrateDebevec
 Inverse camera response function is extracted for each brightness value by minimizing an objective function as linear system. Objective function is constructed using pixel values on the same position in all images, extra term is added to make the result smoother.
 
class  org.bytedeco.javacpp.opencv_photo.CalibrateRobertson
 Inverse camera response function is extracted for each brightness value by minimizing an objective function as linear system. This algorithm uses all image pixels.
 
class  org.bytedeco.javacpp.opencv_photo.MergeExposures
 The base class algorithms that can merge exposure sequence to a single image.
 
class  org.bytedeco.javacpp.opencv_photo.MergeDebevec
 The resulting HDR image is calculated as weighted average of the exposures considering exposure values and camera response.
 
class  org.bytedeco.javacpp.opencv_photo.MergeMertens
 Pixels are weighted using contrast, saturation and well-exposedness measures, than images are combined using laplacian pyramids.
 
class  org.bytedeco.javacpp.opencv_photo.MergeRobertson
 The resulting HDR image is calculated as weighted average of the exposures considering exposure values and camera response.
